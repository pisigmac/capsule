# Marketing

## Positioning

**Capsule** is the knowledge system for developers who work with AI.

Not a note-taking app. Not a wiki. Not a documentation generator.

A **knowledge atomizer** — breaking down bloated documents into digestible, AI-ready facts.

## Target Audience

1. **Solo developers with 6+ projects** — knowledge scattered across repos
2. **AI power users** — need clean context windows, not 2,000-token READMEs
3. **Consultants** — context-switch between client codebases
4. **Open-source maintainers** — tribal knowledge in issues, not docs
5. **Platform teams** — need to share learnings across product teams

## Key Messages

- "Your README is too long for Claude."
- "One fact. One file. Zero noise."
- "The context window is expensive. Capsule makes it cheap."
- "Knowledge that doesn't rot."

## Launch Strategy

### Phase 1: Developer Community (Week 1-2)
- Hacker News launch
- Reddit r/programming, r/MachineLearning
- Twitter thread: "I replaced my README with 47 capsules"
- Dev.to article: "Atomic Knowledge for AI Agents"

### Phase 2: AI Community (Week 3-4)
- MCP server announcement
- LangChain integration guide
- Claude Code skill submission
- Cursor extension proposal

### Phase 3: Enterprise (Month 2-3)
- Team sharing features
- SSO integration
- Audit logs
- Compliance documentation

## Competitive Landscape

| Tool | Type | Capsule Advantage |
|------|------|-------------------|
| Notion | Wiki | Too heavy, not AI-native |
| Obsidian | Notes | No API, no search composition |
| Logseq | Outliner | Complex graph, steep learning curve |
| GitBook | Docs | Bloated, not atomic |
| projectsmd | Project spec | Single-project only, no cross-linking |
